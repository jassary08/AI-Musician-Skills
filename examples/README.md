# Examples Gallery

Real outputs produced by the skill scripts in this repo — no GPU, no audio, no
model weights. Every file here was generated by running the commands shown
below. Regenerate any of them yourself to verify.

These examples follow one storyline: **"I found the chords, now make them
playable for me."** That second step — turning a chord list into something a
real guitarist at a real skill level can actually play — is what this skill is
for.

---

## 01 · Pop chorus, beginner, no barre

The flagship case. A natural-language request in plain Chinese, turned into an
open-position arrangement a beginner can play, with a capo chosen automatically
so the hard key becomes easy open chords.

```bash
python ../guitar-arrange-skill/scripts/compose_guitar.py \
  --input-json '{"request":"写一段温暖的 C 大调流行副歌，适合初学者吉他弹唱，不要横按"}' \
  --export-dir 01-pop-chorus-beginner \
  --diagram-png-output 01-pop-chorus-beginner/chord-diagrams.png \
  --pretty
```

- Asked for C major, the arranger put **capo 5** and plays as **G major** so
  every chord is an open shape (`G D Em C`).
- No barre chords, because the request said beginner + 不要横按.
- Outputs: `arrangement.json`, `lead_sheet.json`, `practice_sheet.txt`,
  `arrangement.cho` (ChordPro), `voicing_summary.json`, and a chord-diagram PNG.

See `01-pop-chorus-beginner/practice_sheet.txt` for the readable result.

## 02 · 12-bar blues, common shapes, no capo

Style drives the harmony: a blues request generates **dominant-7th** chords
(`Em7 Am7 B7`) over the classic 12-bar form, not plain triads.

```bash
python ../guitar-arrange-skill/scripts/compose_guitar.py \
  --input-json '{"request":"写一段 E 小调经典 blues 十二小节，常用吉他弹法，不用变调夹"}' \
  --export-dir 02-blues-12bar \
  --diagram-png-output 02-blues-12bar/chord-diagrams.png \
  --pretty
```

## 03 · Same progression, difficulty-adaptive

The point most paid tools miss: the *same* song should be voiced differently for
a beginner and an intermediate player. Same input progression (`A E F#m D`),
two skill levels:

```bash
# beginner — avoids barres
python ../guitar-arrange-skill/scripts/arrange_guitar.py \
  --input-json '{"task":"guitar_arrange","key":"A major","style":"pop","user_level":"beginner","goal":"适合弹唱，避免横按","chords":["A","E","F#m","D"]}' --pretty

# intermediate — allows richer/higher voicings
python ../guitar-arrange-skill/scripts/arrange_guitar.py \
  --input-json '{"task":"guitar_arrange","key":"A major","style":"pop","user_level":"intermediate","chords":["A","E","F#m","D"]}' --pretty
```

Both land on **capo 2 → play as G major**, but the fingerings differ:

| Chord | Beginner | Intermediate |
|-------|----------|--------------|
| D     | `xx0232` (open, difficulty 1) | `x54233` (barre, difficulty 4) |
| Em    | `022000` | `022030` |
| C     | `x32010` | `x32030` |

The beginner version stays in open position; the intermediate version uses a
barre D and richer top notes. This is the "fits *my* level" promise made
concrete.

## 04 · Style-aware generation

When no chords are supplied, the requested **style** shapes the generated
harmony. Same key (C major), three styles:

```bash
python ../guitar-arrange-skill/scripts/compose_guitar.py --input-json '{"request":"写一段 C 大调流行 verse，中级吉他手"}' --pretty
python ../guitar-arrange-skill/scripts/compose_guitar.py --input-json '{"request":"写一段 C 大调 blues verse，中级吉他手"}' --pretty
python ../guitar-arrange-skill/scripts/compose_guitar.py --input-json '{"request":"写一段 C 大调 funk verse，中级吉他手"}' --pretty
```

- **pop** → plain triads (`Em C G D`)
- **blues** / **funk** → dominant 7ths (`A7 D7 E7`)

Each result also carries a full `style_guidance` block (harmony principles,
chord vocabulary, voice-leading focus) that an LLM agent can use to reason about
the arrangement.

## Known limitation (honest note)

When you **supply** a fixed progression and ask for a richer style color (e.g.
hand it `C Am F G` and ask for neo-soul m7/maj7/9), the current arranger
**preserves your triads** rather than recoloring them. Style most strongly
affects harmony when chords are **generated** from scratch (examples 02 and 04).
Stronger "recolor-supplied-chords" behavior is on the roadmap — see the repo
README.
