# Voicing DB Organization

The voicing database is organized by how the arranger actually uses the data.

## Runtime Path

1. `source/chords_db_voicings.json`
   - Immutable imported MIT source data from `tombatossals/chords-db`.
   - Contains raw guitar voicing candidates: `symbol`, `frets`, `fingers`, `position`, `barres`, `difficulty`, and source metadata.
   - Do not edit this file by hand.

2. `overlays/commonness_annotations.json`
   - Editable commonness and style overlay.
   - Adds `commonness`, `status`, `styles`, `contexts`, `canonical_rank`, and notes.
   - The arranger reads this file before scoring candidates.

3. `indexes/voicing_layers.json` and `indexes/layer_indexes/*.json`
   - Optional inspection indexes kept with the release resources.
   - Useful for audits and future filtering, but not the source of truth and not generated by a runtime script.

4. `review/`
   - Optional human-review exports and inspection batches.
   - Files here are not read by the arranger unless explicitly wired in.

## Validation

Validate resources:

```bash
python scripts/validate_rules.py
```
