# Voicing DB Organization

The voicing database is organized by how the arranger actually uses the data.

## Runtime Path

1. `source/chords_db_voicings.json`
   - Immutable imported MIT source data from `tombatossals/chords-db`.
   - Contains raw guitar voicing candidates: `symbol`, `frets`, `fingers`, `position`, `barres`, `difficulty`, and source metadata.
   - Do not edit this file by hand.

2. `overlays/commonness_annotations.json`
   - Editable commonness and style overlay.
   - Generated by `scripts/auto_annotate_voicings.py` and refined by the `/annotator` web UI.
   - Adds `commonness`, `status`, `styles`, `contexts`, `canonical_rank`, and notes.
   - The arranger reads this file before scoring candidates.

3. `indexes/voicing_layers.json`
   - Generated inspection index.
   - Groups voicings by musical taxonomy, playability, style fit, voice-leading traits, and review layer.
   - Useful for audits and future filtering, but not the source of truth.

4. `indexes/layer_indexes/*.json`
   - Split sidecar indexes for lighter lookup:
     - `musical_taxonomy.json`
     - `guitar_playability.json`
     - `release_review_pools.json`
     - `style_fit.json`
     - `voice_leading.json`

5. `review/`
   - Optional human-review exports and inspection batches.
   - Files here are not read by the arranger unless explicitly wired in.

## Update Commands

Refresh the commonness overlay:

```bash
python scripts/auto_annotate_voicings.py
```

Refresh generated indexes:

```bash
python scripts/build_voicing_layers.py --pretty
```

Validate resources:

```bash
python scripts/validate_rules.py
```
